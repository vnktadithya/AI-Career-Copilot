# Contributing Guide

## Branching Rules
- Do NOT push to `main`
- Always create a feature branch:
  feature/<name>-<task>

## Workflow
1. Create branch from `main`
2. Commit often with clear messages
3. Push branch
4. Create Pull Request
5. Request review
6. Merge after approval

## Commit Message Format
feat: new feature  
fix: bug fix  
chore: maintenance  
docs: documentation
